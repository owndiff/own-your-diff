# Code of Conduct

OwnDiff is an open source project for developers, maintainers, security reviewers, and AI-coding-agent users. The project works best when people can report problems, disagree, and improve the tool without being harassed or talked down to.

## Expected Behavior

- Be respectful, direct, and constructive.
- Assume good intent, but take responsibility for impact.
- Keep feedback focused on the code, documentation, design, security model, or user experience.
- Welcome contributors with different backgrounds, skill levels, tools, and workflows.
- Handle security and privacy topics carefully. Do not post secrets, private source code, customer data, or local machine details in public discussions.

## Unacceptable Behavior

- Harassment, threats, stalking, intimidation, or sustained disruption.
- Insults, personal attacks, discriminatory language, or sexualized comments.
- Publishing another person's private information without permission.
- Pressuring maintainers or contributors to reveal private data, credentials, customer information, or unpublished security details.
- Repeatedly derailing issues, pull requests, or discussions after maintainers ask for a change in behavior.

## Reporting

If you see behavior that violates this code of conduct, report it to the project maintainers through the GitHub repository.

For sensitive reports that involve vulnerabilities, private data, secrets, abuse, or anything that should not be public, use GitHub private vulnerability reporting:

https://github.com/owndiff/own-your-diff/security/advisories/new

Include enough context for maintainers to understand what happened, but do not include secrets, private repository content, customer data, tokens, or local absolute paths.

## Enforcement

Maintainers may remove comments, close issues or pull requests, limit participation, or block users when behavior harms the project or community. Enforcement should be proportionate, consistent, and focused on restoring a useful working environment.

Examples of maintainer actions include:

- Asking for clarification or a change in tone.
- Editing or removing harmful content.
- Temporarily limiting participation in an issue, pull request, or discussion.
- Blocking a user from the project for repeated or severe violations.

## Scope

This code of conduct applies to project spaces including issues, pull requests, discussions, documentation, demos, release threads, and community support conversations connected to OwnDiff.

It does not replace the project's [Security Policy](SECURITY.md), license, or normal repository review rules.
